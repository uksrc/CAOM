    abstract objectType Observation

    extends CaomEntity

    an observation is a single top-level entry in an astronomy data centre

    Members

    name type mult description
    collection ivoa:string the name of the data collection this observation belongs to
    uri ivoa:anyURI unique logical identifier for this observation (NEW in CAOM-2.5)
    metaRelease ivoa:datetime optional timestamp after which metadata for the observation instance is public
    sequenceNumber ivoa:integer optional a collection-specific sequence number for observations; re-use or reset is collection specific
    type ivoa:string optional the type of observation (FITS OBSTYPE keyword); usually OBJECT for intent = science
    intent ObservationIntentType the intent of the original observer in acquiring this data
    metaReadGroups ivoa:anyURI 0 or more set of groups with read permission on observation metadata (new in 2.4)
    algorithm Algorithm the algorithm or process that created this observation
    telescope Telescope optional the telescope or facility where this observation was created
    instrument Instrument optional the instrument or detector used to acquire the data
    environment Environment optional the environmental conditions at the time of observation
    proposal Proposal optional the science proposal underwhich this observation was created
    target Target optional the intended target of the observation
    targetPosition TargetPosition optional the intended target position for this observation
    requirements Requirements optional the observational requirements specified by the observer or proposal
    planes Plane 0 or more the component planes belonging to this observation
