    objectType Plane

    extends CaomEntity

    a component of an observation that describes one product of the observation

    Members

    name type mult description
    uri ivoa:anyURI optional unique logical identifier for this product; typically made by adding additional path component(s) to the Observation.uri (NEW in CAOM-2.5)
    metaRelease ivoa:datetime optional timestamp after which metadata for the plane is public; this metaRelease timestamp applies to all children of the plane and to artifacts with releaseType=meta
    metaReadGroups ivoa:anyURI 0 or more list of groups (of users) that are allowed to view the metadata of the plane; this is applicable when metaRelease is null or in the future (new in CAOM-2.4)
    dataRelease ivoa:datetime optional timestamp after which data for the plane is public; this dataRelease timestamp applies to all children of the plane and to artifacts with releaseType=data
    dataReadGroups ivoa:anyURI 0 or more list of groups (of users) that are allowed to access the data of the plane; this is applicable when dataRelease is null or in the future (new in CAOM-2.4)
    calibrationLevel CalibrationLevel optional standard classification of the degree to which the data is calibrated
    dataProductType DataProductType optional standard classification of the type of data product; describes the logical data type for the main artifacts
    observable Observable optional description of the sample (pixel) values; (new in CAOM-2.4) In previous versions the observable was assumed to be flux or intensity of EM radiation.
    quality DataQuality optional flag indicating the quality of the data
    metrics Metrics optional collection of measured quantities that describe the content of the data
    position Position optional description of the position(s) included in the data
    energy Energy optional descritpion of the energy(ies) included in the data
    time Time optional description of the time(s) included in the data
    polarization Polarization optional description of the polarization(s) included in the data
    custom CustomAxis optional description of a custom coordinate axis in the data (new in CAOM-2.4); Since different custom coordinate types can be used with different planes, instances of CustomAxis can only be compared sensibly if they have the same coordinate type.
    uv Visibility optional
    provenance Provenance optional description of the provenance of the data
    artifacts Artifact 0 or more the component artifacts belonging to this plane
